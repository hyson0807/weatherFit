### 슬라이드 4: 시스템 구조
**내용**
```
[사용자] → [웹사이트] → [Supabase DB] → [Cron 스케줄러] → [텔레그램 봇] → [사용자]
```
- 웹사이트에서 정보 등록 (이름, 위치, 알림시간)
- DB에 사용자 정보 및 옷 이미지 저장
- Cron이 매일 아침 알림 발송
- 텔레그램으로 날씨 + 옷차림 수신

**대본** (40초)
> "시스템 구조입니다.
> 먼저 사용자가 웹사이트에서 이름, 위치, 알림 시간을 등록합니다.
> 등록된 정보는 Supabase 데이터베이스에 저장됩니다.
> 그리고 사용자는 자신의 옷 사진을 업로드할 수 있습니다.
> 매일 아침 Cron 스케줄러가 해당 시간에 알림을 발송하면,
> 텔레그램을 통해 날씨 정보와 옷차림 추천을 받아볼 수 있습니다."

---

### 슬라이드 5: 주요 기능
**내용**
1. **사용자 등록**: 웹에서 이름, 전화번호, 위치, 알림시간 입력
2. **옷 업로드**: 카테고리별(상의/하의/외투) 옷 사진 등록
3. **텔레그램 알림**: 매일 아침 날씨 + 내 옷장 기반 옷차림 추천

**대본** (40초)
> "주요 기능은 3가지입니다.
> 첫째, 웹사이트에서 사용자 정보를 등록합니다.
> 둘째, 자신의 옷 사진을 카테고리별로 업로드합니다. 상의, 하의, 외투로 구분됩니다.
> 셋째, 설정한 시간에 텔레그램으로 알림을 받습니다.
> 특징은 내가 등록한 옷 중에서 오늘 날씨에 맞는 옷을 추천해준다는 점입니다."

### 슬라이드 6: 기술 스택
**내용**
| 구분 | 기술 |
|------|------|
| Frontend | Next.js, React, Tailwind CSS |
| Backend | Supabase (DB + Storage) |
| 날씨 API | OpenWeatherMap API |
| 알림 | Telegram Bot API |
| 자동화 | Linux Crontab, Shell Script |

**대본** (30초)
> "사용한 기술 스택입니다.
> 프론트엔드는 Next.js와 Tailwind CSS로 구현했습니다.
> 백엔드는 Supabase를 사용해 데이터베이스와 이미지 저장소를 구축했습니다.
> 날씨 정보는 OpenWeatherMap API를 활용했고,
> 알림은 Telegram Bot API로 발송합니다.
> 자동 실행은 리눅스 Crontab으로 스케줄링됩니다."
